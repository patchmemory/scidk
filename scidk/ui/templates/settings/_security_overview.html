TLS Setup Guide for Docker Neo4j

Enable TLS encryption on a Docker-deployed Neo4j instance:

1. Generate a Self-Signed Certificate

mkdir -p /path/to/neo4j/certificates/bolt
openssl req -newkey rsa:2048 -nodes \
  -keyout /path/to/neo4j/certificates/bolt/private.key \
  -x509 -days 365 -out /path/to/neo4j/certificates/bolt/public.crt \
  -subj "/CN=localhost"

2. Configure Neo4j

Add these lines to your neo4j.conf (or pass as Docker environment variables):

dbms.ssl.policy.bolt.enabled=true
dbms.ssl.policy.bolt.base_directory=certificates/bolt
dbms.ssl.policy.bolt.tls_versions=TLSv1.2,TLSv1.3

3. Update SciDK Connection URI

In Settings → Neo4j, change your connection URI from:

bolt://localhost:7687

To:

bolt+s://localhost:7687

4. Verify Connection

Use the connection test in Settings → Neo4j to verify TLS is working. If using a self-signed certificate, you may need to configure trust settings depending on your driver version.

Production Note: Self-signed certificates are acceptable for development and pilot deployments. For production institutional use, replace with a CA-issued certificate from your organization's PKI infrastructure.
